HTML

  1. Elements are types of content on a webpage, the two most important of which are head and body elements.
  2. The head element contains information about the webpage, known as metadata, which helps the browser render correctly..
    • Common tags include: Title, Style, base, link, meta, script, charset, and viewport.
  3. The body element represents the visible content shown to the user, which they can see and interact with.
    • Common tags include H1-6, p, ul ol and li, img, br and a.
  4. Attributes define additional information for an element.
  5. Elements inside of another elements are called Child Elements. The original is now called a Parent element.
  6. Semantic Elements are elements where the name of the element describes the content and the role it plays. An example would be a table element.
  7. Element vs tag: Tags are what is inside of angle brackets, and element is everything in between opening and closing brackets.
  8. Headings are at the top, contain h1-h6 info.
  9. Footer tags are at the bottom, contain info such as copyright, author, ect.

CSS

  1. CSS, or Cascading Style Sheets, defines colors, aesthetics, visual layout and organization.
  2. There are three ways to style a webpage using CSS.
    • Inline, which is to style directly witht he HTML file.
    • Internal CSS Sheets, which is to insert CSS directly into the head section of an HTML file.
    • External CSS files, which is the practice of keeping CSS in a seperate file, then linking to that file in the Doc.
  3. A margin indicates how much space we want around the outside of an element.
  4. A padding indicates how much space we want around the content inside an element.
  5. An important concept for coding is Seperation of concerns, which is the practice of keeping code in different files based on use.
  6. DRY Method - Do Not Repeat Yourself. Keep code simple!
  7. Declerations are used in CSS to provide values. They consist of values and propertys (of that value).

Git

  1. Creating a branch:
    • navigate to correct directory using cd commands
    • check your current git branch with git status command
    • Use git checkout main to commit your staged content as a new commit snapshot
    • git checkout -b "descriptive name" to create a new branch in your github repo.
  2. Closing a branch:
    • git add -A : stages changes to the working directory.
    • git commit -m ["descriptive message"]: commits your staged content as a new commit snapshot.
    • git pull origin main: Pulls the base branch into the feature branch.
    • git push origin "name of branch": pushes up all changes into git.
  3. Additional commands and options can be found in the git and cls cheat sheets.

JavaScript

  1. Javascript is a highend dynamic language which allows users to interact with websites. It offers such benifts such as:
    • Effects, interactivity, data communication, variables, strings, data types, ect.!
  2. Variables are containers which store values.
  3. Comments are a useful function within Java that allows devs to add text to a code which the code will ignore. Used to describe parts of the code to themselves or others.
  4. Control flow is the order in which a computer executes code in a script.
  5. Conditionals are code functions that allow you to test if an expression is true or not. ( EG: if/else)
  6. Functions allow you to package functionality that you wish to resuse.
    • A set of instructions that tell the computer how to preform a certain task.
    • Does not run automatically, only when "called", also known as an event.
    • Example code: "function.makeTea()" - "console.log("Heating up the water!)" - makeTea()"
  7. Events are code structures which "listen" for activity in the browser and run code is response.
